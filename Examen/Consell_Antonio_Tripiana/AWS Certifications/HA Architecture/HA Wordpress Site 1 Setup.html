HA Wordpress Site (1) - Setup

Building a fault tolerant Wordpress site: LAB 1 - Getting Setup

Network Diagram

Setup S3 Buckets

Go to Storage > S3

Create 2 different buckets, one for code and the other for media assets (images)

  • Bucket 1 name: acloudguruwp-code-12351345
  • Bucket 2 name: acloudguruwp-media-123090983

Setup Cloudfront

Go to Networking & Content Delivery > CloudFront

Create a new Cloudfront distribution with the following configuration:

  • Step 1: Select delivery method - Choose Web distribution.
  • Step 2: Create distribution - select Origin domain name → (bucket 2) → acloudguruwp-media-123090983. Leave everything else as default and click to create distribution.

Setup Network

First let's create a security group for the instances:

Go to Networking & Content Delivery > VPC > Security > Security Groups

  • Create new security group allowing http 80 to everyone, and ssh to everyone.

Now let's create a new security group for the RDS (called rds-launch-wizard), allowing traffic from the security group above (inboud rule, MYSQL/Aurora TCP 3306 with source Security group above.

Setup RDS Database

Now let's create our RDS databse:

  • Go to Database > RDS
  • Click on Create Database
    • Engine options → MySQL
    • Templates → Dev
    • Settings
      • DB instance identifier: acloudguru (or whatever)
      • Master username: acloudguru
      • Master password: ...
    • DB instance size: burstable classes → db.t2.micro
    • Storage: as default
    • Availability & durability → Multi-AZ deployment → Create a standby instance
    • Connectivity: VPC → default VPC
    • Additional connectivity configuration:
      • Subnet group as default
      • Publicly accessible: No
      • VPC security Group: select the rds-launch-wizard security group (created before).
    • Additional configuration:
      • Initial database name: acloudguru
      • Everything else as default, except monitoring that we don't want (disable)
    • CREATE DATABASE!

Setup IAM

Create a new role that allows us to talk to S3:

  • Go to Security, Identity & Compliance > IAM > Roles
  • Create role
    • Service that will use this role: EC2
    • Attach permissions policy: choose Amazon S3 Full access
    • Next
    • Role name: S3ForWP
    • Create role

Setup EC2 instance

Let's create our EC2 instance:

  • Go to Compute > EC2
  • Launch instance
    • Step 1: Choose AMI: Amazon Linux 2 AMI
    • Step 2: Instance Type: t2.micro
    • Step 3: change IAM role to S3ForWP and on advanced details → User data, paste the following bootstrap script:
      • installs apache (httpd), php and php-mysql
      • Create a file for our health checks (healthy.html)
      • Get the latest version of wordpress and installs it and deletes the downloaded directories,
      • Put the correct permissions,
      • Sets htaccess, that will allow us to do our URL rewrite. So it will allow us to serve content out of cloudfront rather than doing it out of our S3 buckets
      • Make sure that our apache will start back up in case EC2 instances restart. 
      #!/bin/bash
yum update -y
yum install httpd php php-mysql -y
cd /var/www/html
echo "healthy" > healthy.html
wget https://wordpress.org/wordpress-5.1.1.tar.gz
tar -xzf wordpress-5.1.1.tar.gz
cp -r wordpress/* /var/www/html/
rm -rf wordpress
rm -rf wordpress-5.1.1.tar.gz
chmod -R 755 wp-content
chown -R apache:apache wp-content
wget https://s3.amazonaws.com/bucketforwordpresslab-donotdelete/htaccess.txt
mv htaccess.txt .htaccess
chkconfig httpd on
service httpd start
    • Step 4: add general purpose storage (default)
    • Step 5: add tag —> Name: MyGoldenImage
    • Step 6: Configure security Group: WebDMZ
    • Launch!

Wait until everything is deployed correctly!
