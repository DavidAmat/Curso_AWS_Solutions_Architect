Autoscaling Groups & Launch Configurations - LAB

Prerequisite:

  1. Delete all instances that you have previously used, as well as ELB

Setup Launch Configuration

In order to create AutoScaling Groups, you need a Launch Configuration:

  1. Go to EC2 > Auto Scaling > Launch Configurations > Create launch configuration
  1. Step 1 - Choose AMI: Linux 2 AMI (or whatever you want),
  1. Step 2 - Choose instance type: t2-micro (or whatever you need).
  1. Step 3 - Configure Type. IAM role will be used for all EC2 instances. User Data will be used as bootstrap script for all new EC2 instances launched.
  1. Step 4 - As default
  1. Step 5 - Configure Security Group - WebDMZ (open to internet)
  1. Create or go to create an autoscaling group using this launch configuration (this is next step explained)

Setup Autoscaling Group (ASG)

Ok, we have created our Launch Configuration before, but this doesn't do anything, it's just a configuration. We need our Auto Scaling group that uses this Launch Configuration. We will create an Auto Scaling group that starts in 3 instances and scales up to 6 instances, randomly across 3 availability zones (ASG is intelligent to put 2 instances in each AZ). Scale will start when >80% CPU utilization is found.

So let's go and configure it:

  1. Go to EC2 > Auto Scaling > Auto Scaling Group > Create Auto Scaling Group

Simulating a Failover

Now let's simulate a failover and see what happens.

  1. Select 2 of your EC2 instances and terminate them.
  1. After 60 seconds (as you put in your ASG scaling policies), you will see new instances starting. This is autoscaling running new instances on its own! :)

You can also ssh and do an infinite loop on some instances (so CPU goes up to 100%). You will see that the same happens.
