Connect to our Custom VPC instances

Connect to Public Instance

Connecting to public instances via SSH is straightforward

ssh -i <your-key.pem> ec2-user@<public-ip>

Connect to Private Instance

Connecting to a private instance that does not have access to internet and does not have public ip address is not that easy. These instances only private ip address.

If we connect to our public instance via ssh and try to connect from our public instance to our private instance via ssh again, it won't work (if you followed the last tutorial). This is because, per default, security groups cannot talk to each other. As public and private instances are in different security groups, thus is not possible for them to connect.

So first, create a new security group for your private instance in the EC2 section, allowing to connect from public subnet (10.0.1.0)

  • EC2 > Security Groups > Create Security Group
  • Allow your desired inbound traffic (see figure)

Attach your private instance to the new security group:

EC2 > Instances > select private Instance > Actions > Networking > Change Security Groups

  • Add the new security group and remove the default one.

Ok we are done. Now ssh into your public instance as before:

ssh -i <your-key.pem> ec2-user@<public-ip>

Once you are in, copy the private ip address of your private instance and try to PING from your public instance to your private instance:

ping <private-ip>

Also ssh will work, but you will need to copy your private key file from your computer to your ec2 (not recommended at all!). For this is better to use ssh tunneling through our public instance, using our public instance as bastion host.
